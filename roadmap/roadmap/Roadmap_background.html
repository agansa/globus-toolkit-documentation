Background

Globus Software

Globus software development began in 1996 as part of an effort to provide the scientific community with a set of tools and services for building distributed applications.

The Globus User Community

One useful way of characterizing Globus users is by their level of interaction with Globus software:

  • Infrastructure builders, whose primary focus is in using the Globus software base as a means of constructing shared infrastructure for use by one of the other identified community members. Infrastructure builders are strongly driven by manageability, configuration, deployment, and operational issues.
  • Application developers, who build domain-specific applications using Globus software.
  • Service developers, who use Globus tools, libraries and runtime services to build new Grid services and capabilities. Infrastructure created by service developers is often more general-purpose than that of application developers.
  • End users, who develop no code but interact with Globus-based infrastructure, often without realizing they are using it.

The Globus Development Community

Globus developers are among the most talented of the computing community. Hailing from many different countries and institutions, Globus developers are scattered around world. Because the community is large and distributed, it does not share a common physical space. Instead, Globus development takes place over the internet, facilitated by an Apache-like forum called dev.globus. The day-to-day operations of Globus development are conducted in an open fashion, using the source repositories, archived email lists, issue tracking mechanisms and web hosting at http://dev.globus.org.

The Purpose of this Roadmap

The purpose of this document is to present the workplans of Globus developers so that the community might provide feedback. The astute reader will note that each work item is represented as its own entry in the Globus bugzilla. All are invited to join in the Globus planning process by adding comments and suggestions to the Roadmap entries that interest them. Community members are also welcome to propose new roadmap items.
