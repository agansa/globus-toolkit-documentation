What do the Details, History, and Overview links mean?

The Details link displays the specific service data associated with the chosen service or monitored resource, associated with a specific GMT timestamp.

  • If you click this link from the Live Overview page, you will see the live service data from the most recent report.
  • If you click this link from any History page, you will see the service data from that point in history.

The History link takes you to a page on which you see, initially, the most recent 6 hours of state associated with the service or monitored resource you selected. On this page you can adjust the time frame and requery to see a longer or shorter period.

The Overview link, which is available from both the Detail and History pages, gives you the state of all monitored services and resources associated with the time you chose.

  • If you click this link from a Detail page, and that detail is from time T, then you will see the state of all services at time T.
  • If you click this link from a History page, then depending upon which row/timestamp you select, you will see the state of all services at that same time.

When I query for history, why does it time out?

If you've asked for history over a significant period of time, especially if the data is reported often, you are triggering a query that needs to return a very large amount of data. This may take a long time, and, you might get various results, including partial data, or no data, depending upon the vagaries of the Java heap. Try using a shorter, more targeted time period.

Why are there duplicate rows?

This happens when reported data have overlapping GoodFrom/GoodUntil values. For example, if a service reports at 13:00 with GoodUntil 13:45, and then reports again at 13:30 with GoodUntil of 14:15, a query for time 13:35 will return both rows.
