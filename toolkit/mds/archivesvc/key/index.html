Overview

The Archive Service (AS) is much like the Index Service in that it is a Grid service which allows you to collect service data from other Grid services. However, the Index Service only aggregates and publishes current data from these subscriptions, constantly replacing existing service data with new service data. In contrast, the AS is a Grid Service designed to preserve service data history.  The AS accumulates service data values over time and, according to various optional storage management strategies, stores the archive in a persistent Xindice database and allows time-based queries of this service data via Xpath.

The AS is currently only compatible with the Globus Toolkit 3.2. We are developing a version that will be compatible with the upcoming 4.0 release.

Features

The Archive Service features:

  • a history of SDEs to the users by saving notification messages from SDEs into a persistent database
  • Xpath query capabilities to retrieve the history from the database
  • indexing special timestamps for faster time-related queries.
