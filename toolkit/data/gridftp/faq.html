Q: I just installed GridFTP but the performance is no better than FTP.

A: One of the most common problems why users do not initially see the performance advantage with GridFTP is because the default TCP buffer size was used. To ensure maximum performance it is highly recommended to set the TCP Buffer size to the bandwidth delay product: 

[Buffer size (KB) = Bandwidth (Mbs) * Round trip delay timeï¿½(ms) / 8]
Another plan is to use parallel streams. See Performance Tuning of GridFTP.

A rough heuristic guideline for the buffer is as follows: 

        Distance between transfer sites         Recommended Buffer Size
                10 miles                           64 Kbytes (Default)
                100 miles                          0.5 MByte
                300 miles                          1.5 MByte
                1,000 miles                        5 MByte
                3,000 miles                        12 MByte
                10,000 miles                       25 MByte

Larger buffer sizes can be used, but these use additional TCP buffer space that may be limited on systems where many simultaneous transfers are occurring.
