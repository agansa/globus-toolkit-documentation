The Globus Toolkit follows a release strategy similar to the one used in the Linux kernel model. The Globus Toolkit is released with a version number which consists of three parts in the following format:

<major>.<minor>.<incremental>

The major release number represents a major state in the Globus Toolkit. It is incremented when the Globus Toolkit undergoes extensive change relative to its last major release.

The minor release number represents a state in the Globus Toolkit in which components have changed enough to create incompatibilities yet not drastically enough to warrant another major release.

The incremental release number changes each time a component of the toolkit is updated. Incremental releases are generally due to bug fixes or small enhancements and occur frequently.

Stable and Development Releases

There are both a stable and a development release of the Globus Toolkit. The minor version number of a particular release is determined by whether the release is stable or development.

Stable releases of the Globus Toolkit have an even minor version number, while development releases of the toolkit have an odd minor version number. For example,

Stable Releases Development Releases
globus-2.0.0 globus-2.1.0
globus-2.0.1 globus-2.1.1
globus-2.0.2 globus-2.1.2

The Globus Project only supports the latest, minor stable release. Since there no new development in a stable release, only bug fixes and small enhancements are given priority and attended to promptly.

The development release does not receive the same level of support from the Globus Project as the stable release receives. For the development release, new features are given priority over bug fixes. Any bugs which are fixed in a development release, however, will be ported back to the stable release if necessary.

Previous stable and development releases are not supported by the Globus Project but they will remain available in the FTP archive.
