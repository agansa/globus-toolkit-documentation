jGlobus 2.0.0 Release

The primary highlight of this release is the usage of JSSE as the security library instead of the pureTLS as has been historically used.

Key goals of this release

  • Package naming stay the same.
  • Classes will be ported just to be backwards-compatible which will be deprecated eventually.
  • Deprecated classes will be replaced by new functionality
  • The minimal required Java version will be 1.5
