This information is for a release that is no longer supported by the Globus Toolkit.

The Globus Toolkit (TM) performs authentication, authorization and other security measures using the GSI (Grid Security Infrastructure).  The GSI is packaged in each of the bundles available for the Globus Toolkit (TM), but can also be downloaded separately here, if only the GSI is needed for your applications. 

Grid Packaging Toolkit

For the GSI, the Grid Packaging Toolkit must first be installed on your system.  To install it, download it from the link below, set GLOBUS_LOCATION in your environment, and run 'build_gpt'.  For additional help, please see the install guide.

[HTTP download] [FTP download]

Source Download

The source bundle contains the source code required for building the libraries and GSI executables (such as grid-proxy-init).  Download the source bundle if you are planning on compiling other applications on top of the GSI.  

[HTTP Download] [FTP Download]

For help on installing these bundles, please see the install guide.

Binary Download

The binary SDK for a bundle contains pre-built, platform specific libraries and headers required to compile and link your programs against. The binary Tools bundle contains the GSI programs required to create user and proxy certificates.

If the binary bundle for your operating system or architecture type is not listed here, you will need to download the source version above, and build it yourself.
