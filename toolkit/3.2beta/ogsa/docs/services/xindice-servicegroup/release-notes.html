Xindice-based service group release notes

Globus 3.0 beta

The Xindice-based servicegroup package provides an implementation of a ServiceGroup style registry that uses Xindice for storage.

The main benefits of this are:

  • Reduced memory foot-print
  • Use of Xindice's update and query capability to implement advanced search and update functions.

Note that the service group will not persist registrations across restarts of the hosting environment.

By default, a database URI of xmldb:xindice://db will be used. It is also possible to use xindice in embedded mode, by specifying a database URI of xmldb:xindice-embed://db. The database URI is configured though a global parameter in $GLOBUS_LOCATION/server-config.wsdd. For example: &parameter name="dbURI" value="xmldb:xindice-embed://db"/>
