The following list allows access to code quality tool information by category:

  • Compilers - The compiler can be considered a Code Quality Tool in its generation of warnings.
  • Source Code Analyzers - Analyzes the source code for errors.
  • Code Coverage Analyzers - Analyzes source code to determine unit test coverage.
  • Profilers - Identifies performance bottlenecks.
  • Memory Leak Detectors - Identifies blocks of memory that are not freed.
  • Bounds Checkers - Detects invalid memory access such as touching dealocated memory or overrunning an array.
  • Resource Analyzers - Detects the use of uninitialized resources or failure to release them.
  • Thread Analyzers - Detects thread problems such as deadlocks and race conditions.
  • Bug Pattern Detectors - Detects patterns in machine code that may indicate errors.
  • Code Complexity Detectors - Detects code that is overly complex.
