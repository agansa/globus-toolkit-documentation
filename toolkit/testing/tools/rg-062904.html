We have an active campaign to determine the coverage of our Java unit tests. We know that we need new and revised unit tests for GT4 and it is hoped that Coverage reports will aid tech coordinators and developers to understand where to focus.
 
The first thing is to make sure the report itself is an accurate representation of the current state - please let me know if you see problems. The next step is read them with an eye to covering holes in testing.
 
The ogsa Coverage analysis report is here:
 
    http://www-unix.mcs.anl.gov/~gaffaney/ogsa/coverage/
 
The corresponding JUnit results from "ant generateTestReport" are here
 
    http://www-unix.mcs.anl.gov/~gaffaney/ogsa/junit/
 
There is some info on the Clover site on interpreting the results here:
 
    http://www.thecortex.net/clover/tutorial/tut4.html

p.p.s. More info on Clover if you are interested:
 
The website is here:
 
    http://www.thecortex.net/clover/index.html
 
This is a commercial product but they gave us a license to the Ant version based on our status as an open source project. To set it up clover.jar is copied to the ant directory and build.xml and build-tools.xml needed are modified as described at:
 
    http://www.thecortex.net/clover/tutorial/tut0.html
 
Code is from the trunk:
   
    cvs co ogsa
 
Go to ./ogsa/impl/java
 
There are three steps to building the report: Instrument the code, run the tests and create the report:
 
    ant with.cover all
    ant generateTestReport
    ant clover.html
 
(analysis was run on Linux it should run on Windows also)
