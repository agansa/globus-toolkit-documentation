What are the technical requirements?

Supported platforms:

  • Windows
  • Linux / Intel 32
  • Solaris

Profiled applications can run Java 1.3.0 or higher. At this moment IBM Java is NOT supported, please use Sun Java instead.

If your code uses profiler API it should run Java version 1.4.0 or higher.

To run profiler UI, you need Java 1.4.2 or 5.0. If you do not have one, please download it at http://java.sun.com/j2se/1.5.0/download.html.

Can I profile my J2EE server?

You can automatically enable profiling in a number of popular application servers with help of "Tools | Integrate with J2EE Server..." action.

To enable profiling in a server that "Tools | Integrate with J2EE Server..." does not currently support, please manually perform some simple steps as described in the Help, section "Running applications with profiler".

If you encounter problems configuring particular application and documentation does not give an answer, please contact us at support@yourkit.com. We'll do our best to make it work.

Can I profile applets?

You can profile applets with help of IDE integration.

You can manually enable profiling for applets running inside browser if Java Plugin is used. Please go to the "Java(TM) Plug-in Control Panel" (accessible from Windows "Control Panel") and add -Xrunyjpagent to the "Java Runtime Parameters" field located in the "Advanced" tab.

On Windows, don't forget to add <YJP Home>\bin\win32 to the PATH environment variable before you start the browser.

On Linux or Solaris, don't forget to add <YJP Home>/bin/<your OS> to the LD_LIBRARY_PATH environment variable before you start the browser.

You can also profile applets started with appletviewer command, passing -J-Xrunyjpagent command line parameter.

Can I start profiling automatically and/or capture snapshot on application exit?

Yes.

If you launch profiled application via IntelliJ IDEA plugin, additional launch parameters can be configured in "Run/Debug Configurations" dialog, tab "Startup/Connection".

If you launch profiled application via Eclipse plugin, additional launch parameters can be configured in "Run | Profile...", tab "YourKit Java Profiler".

Also you can manually specify appropriate parameters of -Xrunyjpagent. For details see Help, section "Agent configuration".

Do you have plans to support Mac OS X?

Low profiling overhead and ability to profile on demand are achieved because we use some unique technique. We believe the technique is absolutely legal since we use only standard API, but it seems that Mac OS X JVM does not expect to be used that way, and crashes.

We hope we'll be able to support Mac OS X when Java 5.0 with JVMTI is ported to Mac OS X.

I've got message "java.lang.UnsatisfiedLinkError: no yjpagent in java.library.path"

This known problem can appear when both conditions are met:

  • -Djava.library.path is explicitly specified
  • YourKit Java Profiler is installed to directory with name that contains non-ASCII characters

We will try to fix this issue in the next version.

As a workaround, please re-install YourKit Java Profiler to directory with name containing only ASCII characters.

I've got message "Error occurred during initialization of VM. Could not find -Xrun library: libyjpagent.so"

First, please make sure you have properly added a path to profiler agent library to the LD_LIBRARY_PATH variable:

  • Linux: add <YJP Home>/bin/linux
  • Linux (outdated, compiled with gcc 2.9, e.g. Red Hat 7): add <YJP Home>/bin/linux-gcc2.9
  • Solaris: add <YJP Home>/bin/solaris
E.g. execute
export LD_LIBRARY_PATH=<YJP Home>/bin/platform:$LD_LIBRARY_PATH

Additional information please find in the Help.

If the problem remains, then most likely library cannot be linked. To see if it is the case, please:

  • cd <YJP Home>/bin/platform
  • ldd libyjpagent.so
If ldd shows non satisfied dependences, please try to resolve them by installing missing shared libraries.
