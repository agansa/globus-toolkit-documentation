information

The information itself as structured data. For monitored entities that are WS-Resources, this corresponds to resource properties.

monitored entities

That which is being monitored (for WSRF services, equivalent to WS-Resources). Essentially any entity in a virtual organization (VO) from which an aggregator service can obtain information: for example, a file, a program, a Web service, or another network-enabled service.

information sources

External sources of information about a monitored entity. [what else can we say about them?]

NOTE: something can be both a monitored entity *and* an information source if it collects information about itself (e.g., when a WSRF service provides information about its own resource properties) 

aggregator source

An aggregator source collects monitoring information about a monitored entity from an information source and passes it to an aggregator service (usually an MDS-Index service). More specifically, an aggregator source is a java class that implements the AggregatorSource interface to take information about a monitored entity from an information source and pass it to an aggregator service in XML. The aggregator runs joined to the MDS-Index service and/or as part of the MDS-Index service. There are several kinds of aggregator sources.

Types of aggregator sources

WSRF aggregator source

One kind of aggregator source is used when the monitored entities are WS-Resources. This source provides some of the WS-Resource Properties of the monitored WS-Resource as monitoring information. What appears in the index in this case are some of the WS-Resource Properties of the monitored entities.

Non-WSRF aggregator source

Another kind of aggregator source allows collection of information about a monitored entity from the information source via [Unix executables]. The output of the [Unix executables] is XML monitoring information. What is in that XML is dependent on what the executable does. One (under development at the moment) is for when the monitored entity is a GridFTP server and provides status information about that server as monitoring information.

Built-in aggregator sources

Several aggregator sources are included as part of the GT distribution, including:

  • <poll-source-name>, a WSRF aggregator source which polls WSRF services for resource properties
  • <subscribe-source-name>, a WSRF aggregator source which receives resource properties from WSRF services through subscription/notification
  • <executable-source-name>, a non-WSRF aggregator source which runs an executable to acquire (or generate) information in some XML form.

Relationship between aggregator source, information source and monitored entity

In theory, an aggregator source could simply generate information on its own; in practice, however, aggregator sources generally communicate with information sources. In turn, information sources can publish information about themselves (which also makes it a monitoring entity) but many times publishes information about a separate monitored entity.

The following table uses three different examples to describe the relationships:

Example Aggregator source Information source Monitored entity
  <poll-source-name> The WSRF service providing the actual resource property information. The same WSRF service.
  <subscribe-source-name> The WSRF service providing the actual resource property information. The same WSRF service.
Collecting information about a GridFTP server <executable-source-name> The executable program that acquires the information. The GridFTP server.
