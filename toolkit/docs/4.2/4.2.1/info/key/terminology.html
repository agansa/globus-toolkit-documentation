Aggregator Service

A service that collects information using the WS-MDS aggregation framework, and then does something with it. WS-MDS includes two aggregation services: MDS-Index, which publishes information as service groups, and MDS-Trigger, which runs an arbitrary executable program, passing the information as input to that program. A third aggregation service is planned: and MDS-Archive, which will maintain an archive of historical information.

aggregator source

An aggregator source collects information about a monitored entity from an information source and passes it to an aggregator service (usually an MDS-Index service). More specifically, an aggregator source is a java class that implements the AggregatorSource interface to take information about a monitored entity from an information source and pass it to an aggregator service in XML. The MDS-Index service calls the aggregator to collect information. There are several kinds of aggregator sources.

Several aggregator sources are included as part of the GT distribution, including
  • <poll-source-name>, a WSRF aggregator source which polls WSRF services for resource properties
  • <subscribe-source-name>, a WSRF aggregator source which receives resource properties from WSRF services through subscription/notification
  • <executable-source-name>, a non-WSRF aggregator source which runs an executable to acquire (or generate) information in some XML form.

Relationship between aggregator source, information source and monitored entity

In theory, an aggregator source could simply generate information on its own; in practice, however, aggregator sources generally communicate with information sources. In turn, information sources can publish information about themselves (which also makes it a monitoring entity) but many times publishes information about a separate monitored entity.
