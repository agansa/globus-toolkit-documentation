Features new in release GT 4.0:

  • Create Replicator: user creates a replicator resource by specifying a set of desired files identified by their logical filenames (see RLS) and the target physical filenames to which copies should be transfered.
  • Start Replicator: user starts the replicator resource thus invoking the replication activities which include locating, transfering, and registering new replicas.
  • Replicator Resource Properties: user may get resource properties of the Replicator (refer to schema for details on resource properties of the Replicator).
  • globus-replication-create: command line tool accepts a replication request file, initializes a replication request, delegates the user credential, creates a Replicator resource, and starts the Replicator. See globus-replication-create --help for help. The replication request file is expected to contain tab-delimited rows where the first field is the LFN name that the user wishes to locate and replicate, and the second field is the URL of the destination of the new replica and is also used as the PFN name when registering the new mapping.

Other Supported Features

  • Supports secure transport, secure conversation, and secure message communication as provided by GT 4.0.
