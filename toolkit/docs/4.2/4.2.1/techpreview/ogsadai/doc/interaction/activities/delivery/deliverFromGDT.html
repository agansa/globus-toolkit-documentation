Activity - deliverFromGDT

Activity Type: Delivery

Description

Retrieve data from another OGSA-DAI service which implements the GridDataTransport portType via invocation of its GridDataTransport::GetFully or GridDataTransport::GetBlock operations.

This activity inputs data from a corresponding outputStream activity in the other service.

Connecting deliverFromGDT and outputStream activities

Example

<deliverFromGDT name="retrieve">
  <fromGDT streamId="outputStreamID" mode="full">
    http://host:port/path/DataService
  </fromGDT>
  <toLocal name="dataSink"/>
</deliverFromGDT>

<deliverFromGDT name="retrieve">
  <fromGDT streamId="outputStreamID" mode="block" numberOfBlocks="25">
    http://host:port/path/DataService
  </fromGDT>
  <toLocal name="dataSink"/>
</deliverFromGDT>

Specification

Element deliverFromGDT:

  • Attribute name - unique name for the activity.
  • Element fromGDT (required) - the URL of an OGSA-DAI service which implements the GridDataTransport portType and from which the data is to be retrieved.
    • Attribute streamId (required) - the name of an outputStream activity on this other service.
    • Attribute mode (required) - the data transfer mode. One of:
      • block - the data is transferred in blocks.
      • full - all the data is transferred in one go.
    • Attribute numberOfBlocks (optional - use with mode="block" only) - the number of blocks to transfer in one go. Using this setting can improve performance of the data transfer, as a SOAP message does not have to be generated for every single block. The multiple blocks will be amalgamated into a single block before transport.
    • Message-level security-related attributes (optional):
      • Used to secure the data delivery.
      • If only some attributes are provided then the others will be set to their default values.
      • The client's credentials will be used if available. If not then the credentials of the host upon which the current service is running will be used.
      • Attribute authentication (required) - authentication to be used.
        • privacy (default) - Use GSI Secure Conversation with message privacy. Third-parties will be unable to read or alter the data.
        • integrity - Use GSI Secure Conversation with message integrity. Third-parties will be able to read the data but be unable to alter the data
        • signature - Use GSI XML Signature.
      • Attribute delegation - credential delegation to be used.
        • full - full delegation.
        • limited (default) - limited delegation.
        • none - no delegation.
      • Attribute authorization - authorization to be used.
        • host - host authorization.
        • self - self authorization.
        • none (default) - no authorization.
  • Element toLocal (required) - output stream.
    • Attribute name - the activity's output stream.
