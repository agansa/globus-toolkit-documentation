How to Use an Encrypted OGSA-DAI Role Map Document

To use an encrypted OGSA-DAI role map document you have to provide the password used to encrypt the document to your Web services container.

The simplest way to achieve this is by setting the ogsadai.password system property before starting your Web services container:

  • On UNIX csh enter: 
    $ setenv CATALINA_OPTS -Dogsadai.password=PASSWORD
  • On other versions of UNIX enter: 
    $ export CATALINA_OPTS -Dogsadai.password=PASSWORD
  • On Windows enter: 
    $ set CATALINA_OPTS=-Dogsadai.password=PASSWORD

However, many popular shells store histories of commands that have been entered, in plain text. Allowing passwords to be stored in such a manner is a potential security hazard.

An alternative, if using Apache Tomcat, is to use a routine which prompts for the password if it detects you are using encrypted role map documents. This can be installed by running the OGSA-DAI deployStartup ant target.

  • Execute: 
    $ ant deployStartup
  • The startup script will attempt to detect whether you are using encrypted OGSA-DAI role map documents and will prompt for a password if this is the case.
