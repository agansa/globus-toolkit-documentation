Provide remote GASS service

Synopsis

globus-gass-server [-slturwcoe -p PORT] [-n option]

Description

The globus-gass-server utility allows the user to start up a stand-alone GASS server, to which he can upload or download files from locally accessible filesystems.

When no "Enable" or "Disable" options are specified, the globus-gass-server runs with the default options of -l -t -u -r -w -o -e. By default, the globus_gass_server will output the base URL it is listening on and then remain in the foreground.

Options

-s

Enable silent mode (Don't output server URL).

The globus_gass_server program will make no output unless a GASS client connects to the URL <base URL>/dev/stdout or <base URL>/dev/stderr.

-l

Enable line buffering of file uploads. Multiple writers to the same URL will be line-interleaved.

-t

The server will expand the "~" in URLs of the form <base URL>/~/<filename> to the home directory of the user-id the program is running as. Otherwise, the ~ will be passed on to the file system untouched.

-u

The server will expand the ~USERNAME in URLs of the form <base URL>~USERNAME/<filename> to the home directory of user USERNAME on the machine the GASS server is running on. Otherwise, the ~USERNAME will be passed on to the file system.

-r

Enable read access to the local file systems

-w

Enable write access to the local file system

-c

Attempts to write to the URL
<base URL>/dev/globus_gass_client_shutdown will cause the globus_gass_server to stop accepting new requests, and to terminate when all outstanding requests are serviced.

-o

Attempts to write to the URL <base URL>/dev/stdout will succeed, with the data sent to the standard output of the globus_gass_server program.

-e

Attempts to write to the URL <base URL>/dev/stderr will succeed, with the data sent to the standard error of the globus_gass_server program.

-n option

Disable the specified option

-p port

Start the GASS server using the specified TCP port.
