3. Running Test

3.1. Start the container

Now that users can authorize to the server up. Note that the container does not currently dynamically detect changes to the grid-mapfile. It is needed to restart after edits.

globus$ grid-proxy-init

globus$ ant startContainer -Dservice.port=8080 or

globus$ java org.globus.ogsa.server.ServiceContainer –p 8080

3.2. Submit Gram job based on command line

With the container running, a client can submit a job. The client will need to set CLASSPATH by using setenv.{sh,csh} to use the GramClient.

sjiang$ grid-proxy-init

sjiang$ source setenv.sh

sjiang$ java org.globus.ogsa.impl.base.gram.client.GramClient http://localhost:8080/ogsa/services/base/gram/MasterManagedJobFactoryService schema/base/gram/examples/test.xml

The successful result of the submitted job could be like:

========== Statue Notification ==========

Job Status: StageIn

=========================================

========== Statue Notification ==========

Job Status: active

=========================================

========== Status Notification ==========

Job Status: done

=========================================

3.3. OGSA Service Browser GUI

With the container running, a client can access the services through a graphic user interface – OGSA Service Browser. The client will need to set CLASSPATH by using setenv.{sh,csh} first.

sjiang$ source setenv.sh

Testing the demo client

If you have a binary distribution, and you just want to test and run the demo and samples quickly (without downloading any external tools), do the following:
  1. Start the grid service container by typing: globus-start-container -p 8080 (Starts up a server on port 8080)
  2. Run the demo client by typing: globus-service-browser http://localhost:8080/ogsa/services/core/registry/ContainerRegistryService

For a guide on how to use the Service Browser demo see Samples.
