This page contains the following topics:

Starting the container
After following instructions for installing and configuring the Globus Toolkit 3.2, you should be able to start the standalone container that is automatically installed.
1

Before you start the standalone container, run <to do what?>:

globus$ grid-proxy-init
2

To start the container, run one of the following:

globus$ ant startContainer -Dservice.port=8080

or 

globus$ java org.globus.ogsa.server.ServiceContainer -p 8080

<what's the difference between these commands?>

Testing GRAM via the command line

Now that the container is running, you can submit a job through the GRAM component. You'll need to set the CLASSPATH by running setenv.{sh,csh}. You'll then run the GramClient program and use the URI to a test XML document.

3

To submit a test GRAM job :

globus$ grid-proxy-init
globus$ source setenv.{sh,csh}
globus$ java org.globus.ogsa.impl.base.gram.client.GramClient
http://localhost:8080/ogsa/services/base/gram/MasterManagedJobFactoryService schema/base/gram/examples/test.xml

This test is successful if the output looks something like this:

========== Status Notification ==========
Job Status: StageIn
=========================================
========== Status Notification ==========
Job Status: active
=========================================
========== Status Notification ==========
Job Status: done
=========================================

Testing the demo client

If you have a binary distribution, and you just want to test and run the demo and samples quickly (without downloading any external tools), do the following:
  1. Start the grid service container by typing: globus-start-container -p 8080 (Starts up a server on port 8080)
  2. Run the demo client by typing: globus-service-browser http://localhost:8080/ogsa/services/core/registry/ContainerRegistryService

For a guide on how to use the Service Browser demo see Samples.
