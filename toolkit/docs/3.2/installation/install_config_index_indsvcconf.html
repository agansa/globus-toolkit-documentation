WS Index Service: Index Service Configuration File (index-service-config.xml)

Use the Index Service Configuration file to configure:

  • grid services. (which section does this?)
  • service data providers (core, custom and ported from Pre-WS) to be enabled for each service referencing this configuration file. (see installedProviders)
  • which service data providers will be executed at startup and/or when the configuration file is reread. (see executedProviders)
  • notifications and suscriptions of service data to other service instances, which facilitates service data aggregation. (see aggregatedSubscriptions)
  • the kinds of service data you want to aggregate. (which section does this?)

The following table describes each section of index-service-config.xml in detail:

  • serviceConfiguration
  • installedProviders
  • executedProviders
  • aggregatedSubscriptions
File Summary

Filename: index-service-config.xml

Default location: <gt3-install-location>/etc (installed with GT 3.2)
serviceConfiguration

This section indicates the namespaces (identified by URLs) for OGSI, Service Data Aggregator, Service Data Provider execution, and XML schema. 

These namespaces are required so that the Apache Axis SOAP engine can properly render the XML entries in the file into runtime objects.

The default contents are:

<serviceConfiguration xmlns:ogsi="http://www.gridforum.org/namespaces/2003/03/OGSI"                     xmlns:aggregator="http://www.globus.org/namespaces/2003/04/service_data_aggregator"
xmlns:provider-exec="http://www.globus.org/namespaces/2003/04/service_data_provider_execution"
xmlns:xsd="http://www.w3.org/2001/XMLSchema">

xmlns:ogis Namespace of the Grid Service
xmlns:aggregator Namespace of the Service Data Aggregator
xmlns:provider-exec Namespace of the ServiceDataProviderExecution
xmlns:xsd Namespace of the XML schema
installedProviders

This section specifies the core Service Data Providers, as well as any Java custom or ported Service Data Providers. 

See Service Data Providers for more detailed information.

These providers are executed when the Index Service activates, based on the execution parameters specified below.

The default contents are:

<installedProviders>
    <providerEntry
class="org.globus.ogsa.impl.base.providers.servicedata.impl.SimpleSystemInformationProvider" />
    <providerEntry class="org.globus.ogsa.impl.base.providers.servicedata.impl.
      AsyncDocumentProvider" />
    <providerEntry class="org.globus.ogsa.impl.base.providers.servicedata.impl.
      ScriptExecutionProvider" />
    <providerEntry class="org.globus.ogsa.impl.base.providers.servicedata.impl.
      HostScriptProvider" />
  </installedProviders>

providerEntry class Required. Specifies a path to the class. The class attribute contains a Java class name that implements one of the supplied Service Data Provider interfaces.
handler (?) need some clarification Optional. Indicates a user-supplied custom callback method where the resulting data will be sent for any post-processing.  See the Custom Data Handlers section of Creating New GT3.2 Service Data Providers for details.
executedProviders

This section indicates which Service Data Providers listed in installed Providers will be executed at startup/reread. One or more piece(s) of service data is produced by each execution of each Service Data Provider specified in this section. Each execution of a Service Data Provider here may use the same individual provider in the installedProviders section multiple times, such as the ScriptExecutionProvider.

Every Java Service Data Provider specified in this section must also be specified in the installedProviders section (described above) of the configuration file; MDS2-style Unix information providers are excepted.

The default contents are:

<executedProviders>
  <provider-exec:ServiceDataProviderExecution>
   <provider-exec:serviceDataProviderName>SystemInformation</provider-
     exec:serviceDataProviderName>
   <provider- 
     exec:serviceDataProviderImpl>org.globus.ogsa.impl.base.providers.servicedata.impl.
     SimpleSystemInformation</provider-exec:serviceDataProviderImpl>
   <provider-exec:serviceDataProviderArgs> </provider-exec:serviceDataProviderArgs>
   <provider-exec:serviceDataName xlns:mds="http://glue.base.ogsa.globus.org/ce/1.1">
     mds:Host</provider-exec:serviceDataName>
   <provider-exec:refreshFrequency>60</provider-exec:refreshFrequency>
   <provider-exec:async>true</provider-exec:async>
  </provider-exec:ServiceDataProviderExecution>
</executedProviders>

serviceDataProviderName  
serviceDataProviderImpl  
serviceDataProviderArgs supplies parameters to the Service Data Providers specified in this entry.
serviceDataName

specifies the name of the new service data to create.  If no name is specified, the name will be created from the tag name of the root element of the XML document resulting from the provider execution.  The name may be qualified with a namespace, using standard XML QName syntax.  For example:

<provider-exec:serviceDataName xmlns:ns="http://www.globus.org/example";ns:SystemInformation</provider-exec:serviceDataName>

refreshFrequency Indicates how often (in seconds) to run this Service Data Provider.
async Indicates that the Service Data Provider specified should attempt to run asynchronously if it is capable of doing so.
aggregatedSubscriptions

This section specifies the Grid services to be indexed by the Index Service.  It is in this section where aggregation occurs.

The following example shows the Index Service subscribing to the GRAM MasterForkManagedJobFactory service in order to aggregate information about the Cluster resources available.

The default contents are:

<!-- Sample Subscription Configuration -->
<aggregatedSubscriptions>
  <aggregator:AggregatorSubscription>
    <aggregator:serviceDataName xmlns:ce="http://glue.base.ogsa.globus.org/ce/1.1">ce:Cluster</aggregator:serviceDataName>
<ogsi:source>http://127.0.0.1:8080/ogsa/services/base/gram/MasterForkManagedJobFactoryService
   </ogsi:source>
    <aggregator:lifetime>1200</aggregator:lifetime>
  </aggregator:AggregatorSubscription>
</aggregatedSubscriptions>

serviceDataName specifies the QName of the service data to which the Index Service will subscribe.
source specifies the service to which the subscription will be made.
lifetime specifies the lifetime of the subscription, in seconds.

For more information, see the WS Index Service: System Administrator's Guide.
