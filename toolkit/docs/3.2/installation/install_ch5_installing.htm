Installing the Globus Toolkit 3.2

There are many ways to install the Globus Toolkit: binary vs source, root vs non-root, and core-only vs by component vs full installation. We will attempt to cover all of the issues here.

Root vs non-root

As a security precaution, we recommend installing GT3 as a non-root user. This enables an important feature of the Toolkit: limited access to the Globus environment to protect blah blah.

Tip from IBM: Instead of switching users repeatedly from within one screen, have a separate window open for each user.

Installation Options

Since the Toolkit is a collection of tools that can be used interchangeably with core, you can install the Toolkit in many ways: <list the options>.

The procedure tables for binary and source installation below, takes you through a basic installation of GT 3.2 Core as well as the Index Service, blah blah components. Detailed installation per component is described in the appropriate component System Administrator's Manuals (links will be provided at the end of the procedure).

Binary Installation

as non-root user

GRAM Note: If you plan to run the GT3 GRAM service please note the section on setting up the required setuid scripts below. We recommend that you install as the user that will run the GT3 GRAM service.

untar GT3 binary package

install GT3 binary package

Open the .tar.gz, and run "./install-gt3 /path/to/install". Make sure that ANT_HOME and JAVA_HOME are set, and that 'ant' and 'java' are on your PATH. If you are using JDK 1.3.1, make sure JAAS is in your CLASSPATH before you run the installer.

Don't panic! GPT may spit out some WARNING messages at you. Don't worry about Perl version complaints. It works with 5.6.1, 5.8.0, and 5.005.

Proceed to the Configuration section for information on acquiring certificates and setting up MMJFS. Until you have run "install-gt3-mmjfs", you will still need your installer directory.
