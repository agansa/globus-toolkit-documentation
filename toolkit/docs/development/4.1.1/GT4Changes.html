Post-4.0 Interface Changes Description Justification Implications Status
1. Proxy file specification in security descriptors has been moved to be containted within <credential> element. This change brings in consistency in how credentials are specified in security descriptor, since certificate filenames are also specified within <credential> element. Bug 2535 Security descriptors that container <proxy-file> element, that were compatible with GT 4.0 will not work anymore. The security descriptor files will need to be rewritten with this change. Code committed to trunk

Command line options for clients used for delegation service, globus-credential-delegate and globus-credential-refresh have been reworked to leverage the common client options in Java WS Core client package. Also, file name to store delgated EPR has been made a requried option. This makes these tools consistent with the rest of the toolkit. Bug 2973. Requiring a filename for the EPR eliminates inconsistency in how the delegate and refresh clients behave and default to. Bug 3077 Options that were used for old clients will not work anymore.
