Overview of the purpose and functionality of the GUI

Command line client for RFT transfers: To submit a transfer to the Reliable File Transfer Service and prints out the status of the transfer on the console.

Command and options:

    rft  [-h <host-ip of the container default 8080> -l <lifetime for the resource default 60mins> -z <authorization mechanism can be self or host. default self> ] -f <path to the file that contains list of transfers>
This is a sample transfer file that the command line client will be able to parse, it can also be found in $GLOBUS_LOCATION/share/globus_wsrf_rft_client/ along with other samples for directory transfers and delete:
(Lines starting with # are comments)

#true=binary false=ascii
true
#Block size in bytes
16000
#TCP Buffer size in bytes
16000
#Notpt (No thirdPartyTransfer)
false
#Number of parallel streams
1
#Data Channel Authentication (DCAU)
true
# Concurrency of the request
1
#Grid Subject name of the source gridftp server
/DC=org/DC=doegrids/OU=People/CN=Ravi Madduri 134710
#Grid Subject name of the destination gridftp server
/DC=org/DC=doegrids/OU=People/CN=Ravi Madduri 134710
#Transfer all or none of the transfers
false
#Maximum number of retries
10
#Source/Dest URL Pairs
gsiftp://localhost:5678/tmp/rftTest.tmp
gsiftp://localhost:5678/tmp/rftTest_Done.tmp

Limitations :

This command line client is very dumb and simple and does not do any intelligent parsing of various command line options and the options in the sample transfer file. It works fine if used in the way documented here.

Command line client for RFT delete : This client is used to submit a list of files to be deleted.
Command and options :
  rft-delete  [-h <host-ip of the container default 8080> -l <lifetime for the resource default 60mins> -z <authorization mechanism can be self or host. default self> ] -f <path to the file that contains list of files that need to be deleted>
This is a sample file that the command line client will be able to parse, it can also be found in $GLOBUS_LOCATION/share/globus_wsrf_rft_client/ along with other samples for directory transfers and delete:
(Lines starting with # are comments)
# Subject name (defaults to host subject)
/DC=org/DC=doegrids/OU=People/CN=Ravi Madduri 134710
gsiftp://localhost:5678/tmp/rftTest_Done.tmp
gsiftp://localhost:5678/tmp/rftTest_Done1.tmp
