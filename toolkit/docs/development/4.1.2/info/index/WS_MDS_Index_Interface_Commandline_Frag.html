    • mds-servicegroup-add for WS MDS Index Service

    mds-servicegroup-add creates a set of registrations to a WS-ServiceGroup and periodically renews those registrations. It is intended primarily for registering grid resources to MDS services such as the index and trigger services.

    Registrations are defined in an XML configuration file, which is documented in the aggregator admin guide.

    Command syntax

       mds-servicegroup-add -s http://foo [options] config.xml

    Options and Arguments

    -s http://foo This dummy option is required but ignored. All end point references used by mds-servicegroup-add are read from the configuration file, not the command line.
    config.xml the configuration file

    The other standard toolkit options [TODO -- link to standard options page] are also supported.

    Configuration File Syntax

    The configuration file used by mds-servicegroup-add defines a ServiceGroupRegistrationsobject, which consists of the following:
    • One or more ServiceGroupRegistrationParameters elements, each of which defines one registration to be maintained. A ServiceGroupRegistrationParameters includes the following elements:
      • RegistrantEPR, the EPR of the entity to be registered (the WSRF service to monitor). This is optional; if it is omitted, the value of defaultRegistrantEPR (see below) will be used.
      • RefreshIntervalSeconds, the frequency with which mds-servicegroup-add will refresh registrations.
      • Content, information specific to the aggregator source being used. If the default aggregator source (the poll source) is used, this should be a GetResourcePropertyPollType, GetMultipleResourcePropertiesPollType, or QueryResourcePropertiesPollType element. If the subscription source is used, the content should be an AggregatorSubscriptionType element. If the execution source is used, the content should be an ExecutionPollType element. These types are all defined in $GLOBUS_LOCATION/share/schema/mds/aggregator/aggregator-types.xsd
    • An optional defaultServiceGroupEPR, defining the service group to which all registrations will be made, unless overridden by ServiceGroupRegistrationParameters.
    • An optional defaultRegistrantEPR, defining the service group to which all registrations will be made, unless overridden by ServiceGroupRegistrationParameters.

    The Globus Toolkit distribution includes an example configuration file with registration parameters for several services at $GLOBUS_LOCATION/etc/globus_wsrf_mds_aggregator/example-aggregator-registration.xml.

    Limitations

    It is necessary for the tool to continue to run in order for the registrations that it maintains to be kept alive, as registrations will otherwise time out.
