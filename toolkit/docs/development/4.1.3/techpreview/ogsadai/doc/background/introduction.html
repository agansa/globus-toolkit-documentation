Introduction

OGSA-DAI is a middleware product that allows data resources, such as relational or XML databases, to be accessed in Grids. Furthermore, it provides some of the base capabilities that would be required to run higher-level data integration services in Grids.

Motivation

OGSA-DAI is motivated by the need to:

  • Provide an extensible framework to easily incorporate data resources on to Grids.
  • Allow metadata about data and the data resources in which they are stored to be obtained.
  • Provide a framework to facilitate the discovery of data that may be distributed, or replicated, over many different types of data resources, the locations of which may not be known beforehand.
  • Allow different types of data models, e.g. XML, relational or files, from distributed data resources to be easily integrated to Grid applications.
  • Allow services to be discovered that provide access to data resources holding required data.
  • Allow data to be accessed through uniform interfaces.
  • Facilitate the integration of data from various sources to obtain the required information.

What Does OGSA-DAI Provide?

OGSA-DAI provides an extensible framework that allows access to and the updating of data resources. Intrinsic data resource capabilities may be exploited and exposed to a Grid, such as performing SQL queries on data stored in relational data resources or XPath statements on XML data, as well as allowing additional data functionality, such as asynchronous delivery or data transformation, to be provided at the service layer. Using the OGSA-DAI interfaces disparate, heterogeneous data resources may be accessed in a uniform manner. It is important to note though that the underlying data model is not hidden - a relational data resource remains a relational resource and will not masquerade as an XML data resource unless that mode of access is already supported by that data resource nor does OGSA-DAI provide a query language that is not already supported by the data resource or through some other third party application.

OGSA-DAI exposes data resources through services. Clients interact indirectly with data resources via these services. OGSA-DAI provides services compliant with three popular Web and Grid services specifications:

  • Web Services Inter-operability (WS-I)
    • http://www.ws-i.org/
    The WS-I version of the OGSA-DAI distribution uses and is tested on a specific version of the UK OMII's web service infrastructure distribution. For more details see: www.omii.ac.uk.
  • Web Services Resource Framework (WS-RF)
    • http://www.oasis-open.org/committees/tc_home.php?wg_abbrev=wsrf
    The WS-RF version of the OGSA-DAI distribution uses and is tested on a specific version of the Globus Toolkit's WS-RF implementation. For more details see GT3.9.* or GT4.* at: www-unix.globus.org/toolkit.
  • Open Grid Services Infrastructure (OGSI)
    • http://forge.gridforum.org/projects/ogsi-wg
    The OGSI version of the OGSA-DAI distribution uses and is tested on a specific version of the Globus Toolkit's OGSI implementation. For more details see GT3.2.* or earlier versions at: www-unix.globus.org/toolkit.

Note that you will have downloaded a specific distribution of OGSA-DAI that conforms to only one of the above standards and uses a specific implementation of it. You must consult the installation notes to find out the exact requirements. Also, these different implementations of OGSA-DAI do not interoperate.

OGSA-DAI services can be used as the basic primitives for creating sophisticated higher-level services that offer capabilities such as data federation and distributed query processing. The OGSA-DAI middleware layer can abstract away concerns such concerns as database driver technology, data formatting techniques and delivery mechanisms, etc from these higher-level services.

OGSA-DAI is intended to contribute to a future in which, it is envisaged, scientists will move away from technical issues such as handling data location, data structure, data transfer and integration and instead focus on application-specific data analysis and processing.

OGSA-DAI and DAIS

The DAIS - Database Access and Integration Services - Working Group of the Global Grid Forum (GGF) is formulating standards for database access and integration services.

OGSA-DAI is involved in this activity and it is intended that OGSA-DAI will eventually provide a reference implementations of these standards. The GGF requires two independent and inter-operable implementations of the standards for the standards to become GGF recommendations.

Further information on the GGF and DAIS are available from:

  • DAIS Working Group:
    • http://forge.gridforum.org/projects/dais-wg/.
  • Global Grid Forum:
    • http://www.gridforum.org/

Note: the current distribution of OGSA-DAI is based on a version of the DAIS specification released for GGF7 - it is not an implementation of the current set of DAIS specifications.
