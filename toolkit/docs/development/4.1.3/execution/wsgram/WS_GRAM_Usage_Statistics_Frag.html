The following usage statistics are sent by default in a UDP packet at the end of each job (i.e. when Done or Failed state is entered).

  • job creation timestamp
  • scheduler type (Fork, PBS, LSF, Condor, etc...)
  • jobCredentialEndpoint present in RSL flag (to determine if server-side user proxies are being used)
  • fileStageIn present in RSL flag
  • fileStageOut present in RSL flag
  • fileCleanUp present in RSL flag
  • CleanUp-Hold requested flag (to determine if streaming is being used)
  • job type (Single, Multiple, MPI, or Condor)
  • gt2 error code if job failed
  • fault class name if job failed (to determine general classes of common faults users experience)

We have made a concerted effort to collect only data that is not too intrusive or private, and yet still provides us with information that will help improve the GRAM component. Nevertheless, if you wish to disable this feature, please see the core admin guide section on Usage Statistics for instructions. There is also a standard disclaimer that is recommended reading.
