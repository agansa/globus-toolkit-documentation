Activity - xUpdateStatement

Activity Type: XML

Description

Run an XUpdate update on an XMLDB data resource. The update can be run upon an individual resource (in an XMLDB a resource is an XML document) within a collection or over an entire collection of resources.

Example

<xUpdateStatement name="statement">
  <collection>musicians/folksingers</collection>
  <resourceId>someResource</resourceId>
  <expression>
    <xu:modifications version="1.0"
      xmlns:xu="http://www.xmldb.org/xupdate">
      <xu:update select="/entry/phone">123123123123</xu:update>
    </xu:modifications>
  </expression>
  <outputStream name="statementOutput"/>
</xUpdateStatement>

Specification

Element xUpdateStatement:

  • Attribute name - unique name for the activity.
  • Element collection (zero or one) - contains the name of the collection that the update will be performed within.
    • Relative to the data resource's base collection.
    • For example, if a data resource is configured for the db/littleblackbook collection then the example would perform the update within the db/littleblackbook/musicians/folksingers collection.
    • If this element is omitted then the update will be performed on the base collection for the data resource.
  • Element resourceId (zero or one):
    • Contains the name of the resource in the selected collection which which the update will be run.
    • If omitted then the update will be run over all the resources in the selected collection.
  • Element expression (required) - an XUpdate expression.
    • The XUpdate statement is contained within the element.
    • OR
    • Attribute from - input stream providing the XUpdate statement.
  • outputStream (required) - output stream.
    • Attribute name - name of the activity's output stream.

XML Schema
