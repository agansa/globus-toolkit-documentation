Introduction

The main user issues for this component deal with configuring credential related settings. There are multiple mechanisms for  doing this:

  • Command line options (these are application specific)
  • CoG properties
  • Environment variables
  • Relying on default behavior. The only default behaviors available concern the proxy file and trusted certificates locations.
More information on these mechanisms can be found in the public interface guide.

Command-line tools

This component does not have any associated command line tools.

Graphical user interfaces

This component does not have any associated graphical user interfaces.

Troubleshooting

The main problems a user may encounter are going to be related to credential acquisition and trusted certificate authorities configuration:

  • Credentials could not be acquired. This may happen for a variety of reasons:
    • The configured location of the credentials is wrong
    • The ownership of the credentials is wrong. They need to be owned by the user trying to acquire them
    • The permissions on the credentials are wrong. The private key and implicitly the proxy file need to be readable only by the owner and may not be a link.
  • The certificate the peer presented was not trusted. This may happen for the following reasons:
    • The CA certificate that issued the peer's certificate is not installed in the configured trusted, i.e. CA,  certificates location.
    • The CA certificate that issued the peer's certificate is not readable by the current user
