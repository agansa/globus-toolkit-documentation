Programming Model Overview

This component consists abstractly of two services: the managed job factory service (MJFS) and the managed job service (MJS).

In actuality there are three, two of which implement the basic port type of the MJS. The first one is the service which actually talks to a particular local resource manager to execute a process on the remote computer or cluster. This one is called a managed executable job service (MEJS). The second is a special implementation which accepts a multi-job description, breaks the description up into single-job descriptions, and then submits each of these so-called "sub-jobs" to an MEJS. This implementation is called the managed multi-job service (MMJS).

Because of the fact that these two services use the same port type, the API for accessing both the MEJS and the MMJS are identical. The MJFS creates the appropriate resource depending on the factory type--there is a special "Multi" factory for mutli-jobs. The appropriate job description type is required for the two different classes of managed job.
