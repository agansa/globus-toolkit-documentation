Globus Toolkit 3.9.2 Documentation: GridFTP

Command line tool: globus-url-copy

This client is a basic URL-to-URL copy. In the 3.9.2 release, it now includes a -stripe option for striped transfers.

Note: Command line options and configuration file options may both be used but the command line overrides the config file.

Arguments

globus-url-copy [optional command line switches] Source_URL Destination_URL

where:

[optional command line switches]
 See Options below for a list of available options. 
Source_URL

Specifies the original URL of the file(s) to be copied.

If this is a directory, all files within that directory will be copied. 

Destination_URL

Specifies the URL where you want to copy the files.

If you want to copy multiple files, this must be a directory.

Note: Any url specifying a directory must end with /

Limitations

There are numerous known memory leaks.

Threaded builds are not expected to work properly.

Error responses are unclear and inconsistent.

Running as a daemon (-s or -S) sometimes triggers a bug that causes an assertion that does not come up when running as a single process.

The server will leak a socket (and the child process will hang in daemon mode) when the client 1) sends a PASV and then 2) sends RETR or STOR and then 3) terminating.

The server running in inetd mode may not automatically load the configuration file from $GLOBUS_LOCATION/etc/gridftp.conf. Use the -c <configfile> parameter to specify the configuration file.

When running a striped transfer with parallelism of more than 1, the transfer may hang when finished, but before eof. This is more likely as parallelism increases and the size of the transfer decreases.

An assertion or other failure may occur when the connection between a data node and frontend is closed unexpectedly.

There is no security whatsoever on the connection between the frontend and data nodes. Additionally, the data node only operates as the user it is started as.

Options

General options 
-v
 Show server version information. 
-V
 Show version info for all globus packages used by the server. 
-p <port>
 Specify the port to listen on. 
-s, -S
 Run in daemon mode. 
-i

Run in Inetd mode.

Also see Configuring the GridFTP server to run under Xinetd/Inetd. 

-ns

Run with no security at all.

Important: This option allows unencrypted access to anyone. Use this for testing only!

 
-aa

Allow unencrypted anonymous access.

Should be run as an unprivileged user, or as root when the anonymous_user and anonymous_group options are set in the configuration file (see Configuration).

Note: GSI clients may still connect as normal when this mode is enabled.

 
-d <num>

Specify debug level.

1 Only shows errors (default)
3 Shows other information as well.
7 Dumps the entire control channel. 
-l <logfile>

Specify the full path to a logfile.

Logging defaults to stderr. This is the only way to get logging/debug info with -i or -S.

 
-c <config>
 Load this configuration file instead of the default $GLOBUS_LOCATION/etc/gridftp.conf 
-G <globus_loc>
 Use this path as the $GLOBUS_LOCATION 
-bs <num>
 Specify the size of buffers used for reading from disk.
Striping/remote data node options 
-dn
 Run as a remote data-node. 
 -r <host:port>[,<host:port>...]
 Specify list of remote data-nodes to connect to.
