Globus Toolkit 3.9.2 Development Documentation: GridFTP

Known Issues and Limitations of the GridFTP server (globus-gridftp-server)

There are numerous known memory leaks.

Threaded builds are not expected to work properly.

Error responses are unclear and inconsistent.

Running as a daemon (-s or -S) sometimes triggers a bug that causes an assertion that does not come up when running as a single process.

The server will leak a socket (and the child process will hang in daemon mode) when the client 1) sends a PASV and then 2) sends RETR or STOR and then 3) terminating.

The server running in inetd mode may not automatically load the configuration file from $GLOBUS_LOCATION/etc/gridftp.conf. Use the -c <configfile> parameter to specify the configuration file.

When running a striped transfer with parallelism of more than 1, the transfer may hang when finished, but before eof. This is more likely as parallelism increases and the size of the transfer decreases.

An assertion or other failure may occur when the connection between a data node and frontend is closed unexpectedly.
